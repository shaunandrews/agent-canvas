# Product

## Register

product

## Users

Agent Canvas is for builders who want an AI to show its work visually: documents, media, website previews, research boards, generated artifacts, and spatial plans. The first users are developers embedding canvas output in agentic apps, and internal teams prototyping AI workflows that need more than a chat transcript.

## Product Purpose

Agent Canvas provides a reusable React package for an infinite canvas that can be driven by humans or by AI actions. It should make it simple to render rich artifacts, inspect structured canvas state, apply validated operations, and persist or sync snapshots through the host app.

## Brand Personality

Quiet, exact, capable. It should feel like a working surface for thought, not a whiteboard toy or a decorative demo.

## Anti-references

Avoid generic whiteboard chrome, heavy drawing-tool complexity, glossy AI assistant tropes, card-heavy SaaS dashboards, and anything that makes the AI interaction feel magical instead of inspectable.

## Design Principles

1. Structure first: every visible thing should have a simple record an AI can read and modify.
2. Canvas as output, not only drawing: documents, embeds, media, and generated previews are first-class.
3. Keep host apps in control: package APIs should be composable and not prescribe storage or agent runtime.
4. Make spatial context portable: expose snapshots and visible-region summaries for model input.
5. Start small enough to trust, with room for collaboration, custom renderers, and richer tools later.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard focus for selectable nodes, readable controls, reduced-motion support, and predictable behavior across pointer, trackpad, and touch devices.
